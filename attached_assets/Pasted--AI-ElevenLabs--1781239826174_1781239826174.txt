现在 AI 语音已经能播放了，但体验还不对。请只优化语音体验和用户录音，不要改页面视觉、背景、梦境档案、回忆走廊。

这次要解决三个问题：

一、换掉阿暖当前的声音

现在 ElevenLabs 的声音像外国人讲中文，语调怪，不像地道中文聊天。

请不要继续用 Bella 这个声音。

我希望阿暖的声音变成：

* 低沉一点
* 男声或中性偏男声
* 像一个很会倾听的朋友
* 语气平稳
* 不要播音腔
* 不要夸张起伏
* 不要像外国人念中文
* 不要甜美女声
* 不要机器人感
* 更像晚上跟你认真聊天的人

请从当前 ElevenLabs 账号可用 voices 里自动筛选更适合中文的低沉男声/中性声音。

要求：

1. 调用 voices API 获取当前账号可用 voices。
2. 不要使用 free plan 不可用的 library voices。
3. 优先选择中文表现更自然的 voice。
4. 如果找不到明确中文 voice，就选低沉、平稳、自然的 male / neutral voice。
5. 把最终使用的 voice name 和 voiceId 打印到 server log。
6. 如果一个声音中文很怪，方便后续替换 voiceId。

voice settings 建议：

* stability 高一点：0.65 - 0.85
* similarity_boost：0.7 - 0.85
* style 不要太高：0.1 - 0.25
* use_speaker_boost: true

目标：
声音要像真实人低声聊天，不要像外国人念中文。

二、修复用户自己的录音识别

现在用户点击麦克风后，虽然能显示语音卡片和声波，但没有真正识别我的语音。
它还是在使用 demo fallback 文本，比如：
“我梦到自己一直在赶路，但怎么都赶不上。”

这是错误的。

请把用户录音链路修好：

正确流程：
点击麦克风
→ 开始真正录音/语音识别
→ 用户说话
→ 停止录音
→ 得到真实 transcription
→ 把 transcription 作为用户消息发送给 OpenAI
→ OpenAI 根据真实 transcription 回复
→ ElevenLabs 朗读 AI 回复

要求：

1. 优先使用浏览器 SpeechRecognition / webkitSpeechRecognition 做实时转写。
2. lang 设置为 zh-CN。
3. interimResults 可以开启，但最终发送必须用 final transcript。
4. 如果识别到了真实语音文字，就绝对不要使用 demo fallback。
5. 只有在完全没有识别结果、浏览器不支持、权限拒绝时，才使用 fallback 示例。
6. fallback 时要在 console 里明确打印：using fallback transcription。
7. 正常识别时要打印：real transcript = xxx。
8. 停止录音后，输入框/用户消息必须是我真实说的话。
9. 不要每次都自动发送同一句 demo 文本。

验收：
我说“今天晚上梦到一只狗追我”，聊天框里就必须出现这句话，而不是固定 demo 文本。

三、优化文字和语音同步感

现在 AI 回复像是文字先出现，声音后面才读，感觉不同步。

我希望更像：
AI 一边说话，文字一边出现。
用户看着文字逐字出现，同时听到声音，像真实人在开口。

请优化为：

1. OpenAI 回复回来后，先不要立刻把整段文字完整显示。
2. 同时触发：

   * 打字机逐字显示
   * ElevenLabs TTS 生成和播放
3. 如果 TTS 生成需要一点时间，可以先显示状态：
   “阿暖正在开口…”
4. 音频开始播放后，文字继续按自然速度逐字出现。
5. 打字速度要接近语音速度，不要文字一下子跑完。
6. 如果无法精确同步，就至少做到：

   * 语音开始播放时，文字还在逐字出现
   * 不要文字全出来很久后才开始读
7. AI 说话时，在当前消息气泡里显示一个小的“正在说话”状态。
8. 如果可以，在气泡里加一个小光标或闪烁点，像它正在说。
9. 播放失败时，才退回纯文字打字机。

四、增加一个“正在说话字幕板”

为了增强真实对话感，请在 Dream Space 当前角色区域或聊天框上方增加一个轻量字幕板。

当 AI 正在说话时：

* 显示当前人格名字
* 显示正在逐字出现的回复文字
* 像字幕一样慢慢出现
* 不要太大，不要挡住聊天
* 说完后可以淡出，完整消息仍然保留在聊天框里

示例：
阿暖正在说：
“这梦听着就挺堵的。一直赶不上，像脑子半夜还在催你跑。”

视觉：

* 深色毛玻璃
* 当前人格颜色边框
* 柔和发光
* 像梦境里的字幕，不要普通弹窗

五、不要混淆两种语音

用户语音输入：

* 用 SpeechRecognition 转文字
* 目的是让 OpenAI 理解用户说了什么

AI 语音输出：

* 用 ElevenLabs 读 OpenAI 的回复
* 目的是让阿暖开口说话

不要把这两个逻辑混在一起。

六、错误提示优化

如果 ElevenLabs 成功返回音频，就不要显示：
“语音暂时没有接通”。

如果用户录音没有识别到，提示：
“我没听清，可以再说一次。”

如果使用了 fallback，提示：
“当前没有识别到语音，已用示例梦境演示。”

七、不要改动

不要改：

* OpenAI 对话逻辑
* 阿暖 prompt 内容
* Dream Space 主视觉
* Antigravity 背景
* 三人格切换
* 梦境档案
* 保存逻辑
* 回忆走廊
* 星图

这次只修：

1. 换更自然低沉的中文声音
2. 修复用户真实语音识别
3. 让 AI 声音和文字逐字显示尽量同步
4. 增加 AI 正在说话的字幕板

最终效果：
用户真的说一句话，系统识别成真实文字，阿暖根据这句话回复。阿暖的声音低沉自然，像一个人在认真陪你聊天。她说话时，文字同步慢慢出现，而不是先显示一整段再机械朗读。
