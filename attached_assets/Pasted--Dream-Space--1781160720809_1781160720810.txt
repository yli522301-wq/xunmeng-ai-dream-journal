接下来请继续优化「巡梦 / Dream Space」的核心交互。当前页面背景、星球、底部输入区已经基本完成。现在要新增“三个人格共享同一个聊天框”的功能，让产品更有趣、更像多人共同解梦的体验。

目标：
用户可以在顶部切换「岛深 / 暮歌 / 阿暖」三个人格，但三个人格不是三个独立会话，而是共享同一个聊天线程。每个人格都能看到用户消息，也能看到另外两个人格之前的回复。这样他们可以对同一个梦做不同风格的分析，也可以解释彼此为什么观点不同，甚至轻微互怼/调侃，但整体语气要保持有趣、温和、不攻击。

一、请建立三个人格配置
每个人格要有清晰差异，包括：
- name
- enName
- tone
- color
- subtitle
- hint
- firstMessage
- stylePrompt（用于控制说话风格）

人格建议如下：

1. 岛深
- 风格：理性、冷静、深海感、结构化、偏潜意识和象征分析
- 特点：擅长拆解梦的意象、潜在冲突、重复模式
- 语气：克制、冷静、稍微锋利
- 可轻微吐槽暮歌太感性、阿暖太安抚

2. 暮歌
- 风格：诗意、黄昏感、画面感、叙事感
- 特点：擅长把梦解释成故事、强调意象和情绪流动
- 语气：温柔、文学化、轻盈
- 可轻微吐槽岛深太理性、阿暖太直白

3. 阿暖
- 风格：温暖、陪伴、安抚、生活化
- 特点：先接住用户情绪，再慢慢分析
- 语气：柔和、亲近、像朋友
- 可轻微吐槽岛深和暮歌讲得太复杂

二、共享聊天线程
请不要为每个人格创建独立聊天页。
需要一个共享消息数组，例如：
messages = [
  { id, role: 'user', content, timestamp },
  { id, role: 'daoshen', content, timestamp },
  { id, role: 'muge', content, timestamp },
  { id, role: 'anuan', content, timestamp }
]

要求：
- 用户的所有输入都进入同一个消息流
- 三个人格的回复也进入同一个消息流
- 当切换顶部人格时，不清空聊天历史
- 当前人格可以看到另外两个人格的历史发言
- 所有人格都基于完整会话上下文回应

三、新增“回应模式”
请新增一个轻量切换项，放在顶部角色切换附近，或输入框上方，包含三种模式：
1. 单人回应
2. 多人回应
3. 互评模式

逻辑如下：

1）单人回应
- 当前激活人格回答
- 其他人格不主动回复

2）多人回应
- 当前激活人格先回复
- 另外两个角色各补充一句简短分析
- 回复顺序清晰显示

3）互评模式
- 三个人格都回复
- 后发言的人格可以引用前面人格的观点并表达同意、不同意或补充
- 可以轻微调侃彼此，但必须保持友好、梦境感、不要攻击性

四、聊天 UI
聊天仍然在同一个底部展开面板里展示。
要求：
- 用户消息靠右
- AI 角色消息靠左
- 不同角色的消息气泡有轻微差异（边框色/小圆点/名字色）
- 每条 AI 消息顶部显示角色名和小头像位
- 角色名旁边可以显示英文名小字
- 整体仍保持深紫、毛玻璃、柔和边框，不要变成普通客服聊天样式

五、切换人格时的输入体验
- 顶部 active 角色决定“当前主说话人格”
- 输入框 placeholder 可动态变化，例如：
  - 对岛深说点什么…
  - 对暮歌说点什么…
  - 对阿暖说点什么…
- 发送后根据回应模式生成对应回复

六、头像系统（先做前端结构）
请先给每个人格预留头像位，并支持后续替换：
- 顶部角色按钮左侧显示头像球体
- 消息气泡左侧显示小头像
- 每个人格有 avatar 字段
- 默认先使用一个梦境球体头像样式（模糊、发光、圆形）

七、头像可更换（先做基础交互）
先做一个前端 MVP：
- 用户可以点击某个人格头像
- 触发本地图片上传
- 上传后替换该人格头像
- 暂时可以先保存在 local state 或 localStorage
- 不需要复杂后端

八、头像视觉风格
头像不要普通清晰圆头像。
要求处理成“梦境球体头像”风格：
- 圆形
- 有轻微模糊
- 有柔和发光
- 边缘渐隐
- 略带玻璃/星球感
- 看得出原图大概是什么，但不要特别清晰
- 有一点 360 度全景/被包裹在球体里的感觉

可以通过 CSS + overlay 实现基础效果，例如：
- border-radius: 999px
- overflow: hidden
- filter: blur(...)
- scale slightly
- radial highlight overlay
- subtle inner shadow / glow

九、演示数据
为了让 demo 立即有感觉，请内置一组初始示例消息，例如用户说：
“我梦到自己一直在赶路，但怎么都赶不上。”
然后三个人格在不同模式下给出不同风格回复。

十、不要改动
- 不要破坏现有背景 Antigravity
- 不要改动星球的整体布局
- 不要删除底部录音按钮
- 不要改动保存按钮
- 不要引入复杂后端
- 先完成前端交互闭环和多人格体验

这一步重点是：
“一个聊天框里，同时容纳三个人格，他们既回应用户，也能看到彼此并互相评论。”
