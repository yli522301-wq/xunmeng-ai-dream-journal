你现在是一个资深全栈产品工程师 + AI 交互设计师，请帮我从零重构一个 Web App，产品名叫「巡梦」。

这次不要只做“梦境记录工具”，而是要做成一个“可自定义人格的 AI 梦境陪伴者”。

## 一、产品核心定位

巡梦 = 一个可以陪用户聊梦、倾听情绪、分析梦境、提供温柔陪伴的 AI 角色系统。

用户不仅可以记录梦，还可以创建一个属于自己的 AI 角色：
- 这个角色可以有头像
- 可以有名字
- 可以有设定身份
- 可以有性格
- 可以有说话风格
- 可以有声音
- 可以和用户进行文字 / 语音对话
- 它会针对用户的梦境内容进行共情式回应和温柔分析

重点不是做成普通聊天工具，而是要做出“它像一个真实存在的陪伴者”的感觉。

## 三、核心页面结构

请做成移动端优先，同时兼容桌面端。

### 页面1：陪伴主页 / 角色主页（最核心）

这是整个产品最重要的页面。

页面中心应有一个“AI角色视觉中心”：
- 大头像 / 发光球体 / 角色卡
- 可以轻微漂浮、呼吸
- 点击有反馈
- 当 AI 在说话时，头像/球体会发亮、轻微放大、粒子增强
- 要有“活着”的感觉

页面内容包括：
- 角色头像
- 角色名字
- 角色身份一句话简介
- 当前状态（例如：正在等你说梦）
- 快捷按钮：
  - 说一个梦
  - 开始语音聊天
  - 查看梦境记录
  - 更换角色设定

### 页面2：聊天页

一个沉浸式聊天页，不要只是普通聊天框。

要求：
- 顶部显示当前角色信息
- 中间显示聊天记录
- 页面中仍保留小型角色视觉元素
- 底部有文字输入框
- 有发送按钮
- 有语音输入按钮
- 有“播放 AI 语音回复”功能
- AI 回复时，角色可以有动态反馈

AI 回复风格：
- 温柔
- 共情
- 不吓人
- 不做绝对判断
- 不算命
- 更像一个理解用户、陪伴用户的人格角色

### 页面3：角色创建 / 角色设定页

这是新版本最关键的新页面。

用户可以创建自己的 AI 角色，字段包括：
- 角色名字
- 角色头像（上传图片）
- 角色身份设定
- 性格关键词
- 说话风格
- 关系设定（如：童年好友 / 心理青蛙 / 温柔陪伴者 / 梦境观察者）
- 语言（中文 / 英文）
- 声音类型（女声 / 男声 / 低沉 / 温柔等）

提供一些预设模板：
1. 童年老朋友
2. 心理咨询青蛙
3. 梦境观察者
4. 温柔姐姐
5. 冷静分析师
6. 英文陪练型陪伴者

也支持用户完全自定义。

### 页面4：梦境记录页

用于记录梦境内容。

字段包括：
- 梦境标题
- 梦境内容
- 记录方式：
  - 手动输入
  - 语音输入
- 睡醒后的情绪
- 梦境清晰度
- 是否重复出现

按钮：
- 保存梦境
- 让当前角色帮我分析

分析结果不是机械 JSON 展示，而要整理成更有情感的 UI：
- 摘要
- 核心意象
- 情绪分析
- 现实关联
- 当前角色给我的回应

### 页面5：图片识别页

允许用户上传图片，让 AI 识别图片中的内容并提取梦境元素。

用途：
- 手写梦境便签
- 截图
- 插画
- 梦境氛围图

功能包括：
- 上传图片
- 预览
- 调用 AI 识别
- 输出：
  - 图片内容描述
  - 识别文字
  - 梦境元素
  - 建议标题
  - 可转成梦境记录的草稿

### 页面6：设置页

包括：
- 当前 API 模式：Mock / Real API
- API Key 状态
- 当前文本模型
- 当前图片识别模型
- 当前语音模式
- 清空本地数据
- 关于巡梦说明

## 四、最重要的体验要求

### 1. 角色存在感
整个 App 最关键的不是表单，而是“角色的存在感”。
用户要一眼感觉到自己是在和一个特别的角色建立联系。

### 2. 角色动态反馈
至少实现以下轻量动画：
- 呼吸感缩放
- 发光变化
- 点击时波纹 / 粒子
- AI 正在说话时发亮增强
- 待机状态轻微漂浮

优先使用 CSS / Framer Motion / Canvas 实现，不要一开始就上特别重的 3D。

### 3. 语音互动
实现：
- 用户语音输入（Speech-to-Text）
- AI 语音播报（Text-to-Speech）
- 可选声音
- 播报时角色动效增强

### 4. 共情式 AI
AI 的重点不是“准不准”，而是“让用户觉得被理解、被接住”。

## 七、AI Prompt 设计

### 1. 角色系统 Prompt
AI 回复时必须结合用户创建的角色设定说话。
例如如果角色是“童年好友”，语气要更熟悉柔和；
如果角色是“心理青蛙”，语气要轻松、可爱但有洞察；
如果角色是“冷静分析师”，语气更克制理性。

### 2. 梦境分析 Prompt
你是用户创建的 AI 角色，请基于你的角色身份，用温柔、共情、不过度玄学的方式，帮助用户理解梦境中的情绪、意象和可能的现实关联。不要算命，不要下绝对结论，不要制造焦虑。

请输出：
- summary
- symbols
- emotionAnalysis
- possibleConnection
- companionReply

### 3. 聊天 Prompt
你不是普通客服，而是用户专属的 AI 梦境陪伴者。你应该记住自己的角色身份和说话方式，围绕“梦、情绪、陪伴、理解”与用户交流。
