现在视觉和三人格单人共享聊天逻辑已经基本完成。接下来请实现「巡梦」Demo 的核心功能闭环：录音/输入 → 当前人格回复 → 保存梦境。

请不要大改现有视觉，不要改背景 Antigravity，不要改顶部三人格切换结构。重点补功能链路。

一、实现文字输入发送

要求：
1. 用户可以在底部输入框输入文字。
2. 按 Enter 或点击发送按钮后，用户消息加入同一个 messages 数组。
3. 当前 activeCharacter 生成一条回复。
4. 回复也加入同一个 messages 数组。
5. 切换人格时 messages 不清空。
6. 输入发送后清空输入框。
7. 如果输入为空，不要发送。

消息结构建议：
{
  id: string,
  sender: "user" | "daoshen" | "muge" | "anuan",
  content: string,
  timestamp: string
}

二、实现本地 mock AI 回复

暂时不需要接真实 API，先用本地 mock 保证 Demo 跑通。

根据当前 activeCharacter 生成不同风格回复：

1. 岛深：
理性、冷静、结构化、偏潜意识和象征分析。
回复要像：
“我会先看这个梦的结构。你反复提到……这可能不是单纯的情节，而是某种现实压力的象征。”

2. 暮歌：
诗意、画面感、叙事感。
回复要像：
“这个梦像一条被拉长的路。你一直在走，但终点像雾一样往后退。”

3. 阿暖：
温暖、安抚、陪伴。
回复要像：
“先别急着解释它。光是这个梦留下的感觉，就已经很值得被看见了。”

要求：
- mock 回复要参考用户最后一条输入。
- 如果用户提到另一个人格，例如“你怎么看暮歌刚才说的”，当前人格要表现出知道聊天历史。
- 可以简单从 messages 里找到上一条非当前人格的 AI 消息进行引用。
- 不要三个人同时回复，只让当前选中的人格回复。

三、实现录音按钮状态

请给底部麦克风按钮增加录音状态：

状态：
1. idle：未录音
2. recording：录音中
3. processing：处理中 / 转文字中
4. responding：AI 回复中

交互：
- 点击麦克风按钮开始录音，状态变 recording。
- recording 状态按钮要变亮，有轻微 pulse 动画。
- 页面显示提示文案：“正在聆听你的梦…”
- 再次点击麦克风按钮停止录音。
- 停止后把一段模拟识别文字填入输入框或直接发送。

初版可以使用 mock transcription：
“我梦到自己一直在赶路，但怎么都赶不上。”

如果浏览器支持 Web Speech API，可以优先使用 Web Speech API；如果不可用，就 fallback 到 mock transcription。
不要因为语音识别不可用导致页面报错。

四、实现 AI 回复中状态

用户发送消息后：
- 显示一个轻量 loading 状态，例如：
  “岛深正在往梦的深处看…”
  “暮歌正在整理梦的画面…”
  “阿暖正在慢慢听你说…”
- 600ms 到 1000ms 后再显示 mock 回复，模拟真实 AI 思考。
- loading 不要阻塞页面，不要造成布局跳动。

五、实现保存梦境

右上角“保存”按钮要有实际动作。

点击保存时，把当前梦境会话保存到 localStorage。

保存数据结构建议：
{
  id: string,
  title: string,
  createdAt: string,
  updatedAt: string,
  activeCharacter: string,
  messages: messages,
  summary: string,
  mood: string
}

title 生成规则：
- 优先取用户第一条消息的前 12 个字
- 如果没有用户消息，使用“未命名梦境”

summary 生成规则：
- 先用最后一条 AI 回复作为 summary
- 如果没有 AI 回复，就用用户第一条消息

mood 可以先默认：
“朦胧”

六、保存成功反馈

保存成功后显示一个 toast 或轻提示：
“这段梦，已经被收进梦里了。”

提示 2 秒后自动消失。

如果没有任何消息，点击保存时提示：
“先说一点梦的内容，再保存。”

七、保留现有视觉

不要改：

只补：

八、测试场景

请确保下面流程能跑通：

1. 选择暮歌。
2. 输入：“我梦到自己一直在赶路，但怎么都赶不上。”
3. 暮歌回复。
4. 切换到岛深。
5. 输入：“岛深，你怎么看暮歌刚才说的？”
6. 岛深回复，并表现出知道前面暮歌说过话。
7. 点击保存。
8. 出现保存成功提示。
9. 刷新页面后不要报错。
