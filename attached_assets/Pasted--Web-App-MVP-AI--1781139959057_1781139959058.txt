你现在是一个资深全栈产品工程师，请帮我从零开始开发一个名为「巡梦」的 Web App MVP。

「巡梦」是一款 AI 梦境记录与解析工具，用户可以记录自己的梦境，通过文字、语音、图片等方式输入内容，AI 可以帮助用户整理梦境、提取关键词、分析情绪、生成梦境摘要，并支持和 AI 进行语音/文字对话。

请你直接生成一个可以运行的 Web App，不要只写说明文档。优先保证功能闭环、界面美观、结构清晰、后续方便接入真实 API。

## 一、产品定位

产品名：巡梦

核心概念：
用户醒来后，可以快速记录梦境。
AI 帮用户把混乱的梦境整理成清晰的记录。
用户可以和 AI 对话，探索梦境背后的情绪、记忆、压力和灵感。

产品气质：
神秘、安静、治愈、夜晚感、梦境感。
不要做成普通笔记软件。
整体风格可以偏深色、星空、雾气、月光、渐变、玻璃拟态。
但界面必须简洁，不要太复杂。

## 二、核心功能 MVP

请实现以下核心页面和功能：

### 1. 首页 / 梦境仪表盘

首页展示：

* App 名称：巡梦
* 一句副标题：记录梦，理解自己
* 今日快速入口按钮：

  * 记录梦境
  * 和 AI 聊聊
  * 上传图片识别
* 最近梦境列表
* 每条梦境卡片显示：

  * 梦境标题
  * 日期时间
  * 情绪标签
  * 关键词
  * 简短摘要
* 如果没有记录，显示空状态，引导用户创建第一条梦境。

### 2. 新建梦境记录页面

用户可以通过文字记录梦境。

字段包括：

* 梦境标题
* 梦境正文
* 睡醒后的情绪选择：

  * 平静
  * 焦虑
  * 开心
  * 恐惧
  * 迷茫
  * 怀念
  * 奇怪
* 梦境清晰度：

  * 模糊
  * 一般
  * 清晰
* 是否是重复梦境
* 保存按钮
* AI 整理按钮

点击“AI 整理”后：
调用 AI API，将用户输入的梦境内容整理成：

* 梦境摘要
* 关键词
* 情绪分析
* 可能的现实关联
* 一句温柔的回应

如果暂时没有 API Key，也要提供 mock AI 结果，保证页面能完整演示。

### 3. AI 对话页面

实现一个聊天页面，用户可以和 AI 讨论自己的梦。

功能要求：

* 聊天消息列表
* 用户输入框
* 发送按钮
* 支持文字对话
* AI 回复内容围绕梦境、情绪、记忆、灵感进行温柔分析
* 不要把 AI 设定成算命、玄学预测软件
* AI 应该像一个温柔的梦境观察者，帮助用户理解自己

系统提示词建议：

你是「巡梦」里的 AI 梦境观察者。你的任务不是算命，也不是给用户下结论，而是温柔地陪用户回忆梦境、整理情绪、发现可能的现实关联。你的回答要细腻、克制、治愈，不要吓人，不要绝对化判断。

如果没有 API Key，使用 mock 回复。

### 4. 语音输入功能

在记录梦境页面和 AI 对话页面，都加入语音输入按钮。

MVP 要求：

* 使用浏览器 Web Speech API 实现语音转文字
* 点击麦克风按钮开始识别
* 识别结果自动填入输入框
* 显示“正在聆听...”状态
* 如果浏览器不支持语音识别，给出友好提示

注意：
这个功能优先使用 Web Speech API，不需要一开始就接复杂语音模型。

### 5. 图片 / 文字识别页面

实现一个上传图片识别页面。

使用场景：
用户可能会上传手写梦境、截图、便签、梦境相关图片。

功能要求：

* 支持上传图片
* 显示图片预览
* 点击“识别图片内容”
* 如果接入了视觉模型 API，则调用 API 识别图片内容
* 如果没有 API Key，则返回 mock 识别结果
* 识别结果可以一键转成梦境记录

识别结果结构：

* 图片内容描述
* 识别出的文字
* 可能的梦境元素
* 建议标题
* 一键保存为梦境

### 6. 梦境详情页

点击梦境卡片进入详情页。

详情页展示：

* 标题
* 日期
* 原始梦境正文
* AI 整理后的摘要
* 关键词
* 情绪标签
* 情绪分析
* 可能的现实关联
* AI 的温柔回应
* 编辑按钮
* 删除按钮
* 和 AI 继续聊这个梦按钮

### 7. 数据存储

MVP 阶段优先使用本地存储 localStorage，保证不用数据库也能跑通。

请设计清晰的数据结构，例如：

DreamRecord:

* id
* title
* content
* mood
* clarity
* isRecurring
* createdAt
* summary
* keywords
* emotionAnalysis
* possibleConnection
* aiResponse
* imageUrl 可选

后续需要方便迁移到 Supabase / Firebase。

### 8. API 接入方式

请在项目中预留 AI API 接入能力。

要求：

* 使用环境变量保存 API Key
* 不要把 API Key 写死在前端代码里
* 可以使用后端 API route / server function 做中转
* 支持以下 AI 能力：

  * 文本对话
  * 梦境整理
  * 图片识别
* 如果没有配置 API Key，自动 fallback 到 mock 模式

请在代码里清楚标注：

* 哪里填写 API Key
* 哪里切换真实 API / mock
* 哪里修改模型名称

默认可以先兼容 OpenAI 风格接口：

* chat/completions
* 支持文本
* 支持 vision 图片输入

也请尽量让结构方便替换成其他模型 API，例如 DashScope、Gemini、Claude。

## 六、AI 功能的 Prompt 设计

请在代码中内置以下三类 AI Prompt。

### 1. 梦境整理 Prompt

你是「巡梦」的 AI 梦境整理助手。请根据用户输入的梦境内容，帮用户整理梦境。请不要算命，不要做绝对化判断，不要制造恐惧。请用温柔、细腻、克制的语言输出。

请返回 JSON 格式：

{
"summary": "梦境摘要",
"keywords": ["关键词1", "关键词2", "关键词3"],
"emotionAnalysis": "情绪分析",
"possibleConnection": "可能的现实关联",
"aiResponse": "一句温柔回应"
}

### 2. AI 对话 Prompt

你是「巡梦」中的 AI 梦境观察者。你不是算命师，也不是心理医生。你会温柔地陪用户回忆梦境，帮助用户整理情绪、发现梦境中的意象和现实生活的可能联系。你的回答要简洁、细腻、治愈，不要吓人，不要给绝对结论。

### 3. 图片识别 Prompt

请识别这张图片中的内容，尤其关注文字、手写笔记、梦境相关元素、情绪氛围和可能的象征物。请返回：

{
"description": "图片内容描述",
"recognizedText": "识别出的文字",
"dreamElements": ["元素1", "元素2"],
"suggestedTitle": "建议梦境标题",
"draftContent": "可以转成梦境记录的正文"
}
